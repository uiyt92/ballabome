# 모바일 웹 최적화 (360px / 393px 기준) 워크플로

이 워크플로는 모바일 디바이스(주로 360px Android 기기 및 393px iOS 기기)에서 완벽한 가독성과 사용성을 제공하기 위한 UI 최적화 기법을 다룹니다.

## 기본 원칙
모바일에서 레이아웃이 깨지거나 텍스트가 잘리는 것을 방지하기 위해 **기본 클래스는 모바일(360px~393px)에 맞추고, PC(Tablet 이상) 화면은 `md:` (min-width: 768px) 이상 데스크탑 환경에 맞추어 점진적으로 디자인을 추가**합니다.

## 핵심 기법 체크리스트 및 적용 방법

### 1. 반응형 레이아웃 토글 (`hidden`, `block`, `flex`)
모바일과 PC 화면비가 다르기 때문에 불필요한 요소는 감추고 서로 다른 줄바꿈을 적용합니다.
- **예시**: `<br class="hidden md:block">` 
  - *모바일(360px/393px)* 에서는 줄바꿈을 무력화하여 텍스트가 자연스럽게 이어지도록 하고, *PC* 에서는 강제로 줄바꿈합니다.
- **모바일 전용 요소**: `<div class="md:hidden block">...</div>`
  - 모바일 사용자에게만 스와이프 안내 문구 등 특정 요소를 노출합니다.

### 2. 스와이프(가로 스크롤) 캐러셀 레이아웃 적용
세로로 지나치게 길어지는 요소(후기 사진, 리스트 등)는 모바일 화면 비율을 고려해 가로 슬라이드로 변경합니다.
```html
<!-- 부모(컨테이너) 적용 속성 -->
<div class="flex overflow-x-auto snap-x snap-mandatory scrollbar-hide gap-4">
    <!-- 자식(카드) 요소 크기를 화면 뷰포트에 맞게 80~90% 정도로 설정 -->
    <div class="snap-center shrink-0 w-[85%] max-w-[347px]">
        ...
    </div>
</div>
```

### 3. 반응형 폰트 사이즈 및 Padding/Gap 조정
데스크탑의 거대한 폰트와 여백은 360px/393px 디바이스 화면을 뚫고 나가거나 터치를 불편하게 만듭니다.
- **텍스트 크기 변환**: `<h2 class="text-[1.8rem] md:text-[2.8rem] leading-tight">`
  - *모바일* 에서는 `1.8rem`(또는 `2.2rem`) 정도로 한눈에 들어오게 맞추고, 행간(`leading-tight`)을 좁혀 응집력을 높입니다.
- **Padding/Gap 조정**: `<div class="p-6 md:p-14 gap-4 md:gap-8">`
  - 모바일에서는 여백을 조금 타이트하게 잡아 핵심 콘텐츠에 집중시킵니다.

### 4. 1열 ↔ 다열 그리드 자동 변환 (Grid Column)
데스크탑에서 나란히 2~3열로 보이는 요소는 모바일 폭에서 너무 좁아지므로 1열로 떨어뜨립니다.
```html
<!-- Grid 속성 -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-6">
    <!-- 자식 요소들이 모바일에서는 위아래로 하나씩, PC에서는 3개씩 나란히 배치됨 -->
</div>
```

## 유의 사항
- 모든 작업 시 **모바일 렌더링 기준 타겟(360px과 393px)**의 뷰포트를 최우선으로 고려해야 합니다.
- 스크롤바가 생기지 않도록 `body`나 최상단 컨테이너에 `overflow-x-hidden` 처리가 잘 되었는지 점검합니다.
